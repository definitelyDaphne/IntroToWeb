CSS makes the web page more appealing, it gives the plain html pages a style, a personality.
Browsing the internet is more like a delightful experience than simply a place to get information.

Shortly after www and html marked their debut, people started thinking, is there a way to make the web page look more attractive...?

Although some early browsers have own internal style language to format the content, developers had no control over it; therefore, didn't get the chance to harvest the benefit.

Frustrated by the failure of trying to layout webpage in an aethetic way using pure HTML, web professionalist decides to introduce new technology - CSS into the web domain. Before this measure took place, developers have no other way to do proper layout except to misuse table and invisible block divisions to achieve the desired outcome, this approach is less than ideal.

There were a few potential candidates competing for the formal style language place. CSS stands out by providing the distinctive cascading feature. It takes users preference into account, rules with different privilege level competing using some predefined algorithm, to decide which one to display at the end result page.

CSS allows separation between content from presentation, which brought several benefits:
  • efficiency and maintainability
    • html and css both ONLY do what they are suppose to do, and do it well. Developer only have to surround the text in html with proper tags, and later on in CSS define how should each type of tag look like. It saves time of recurring apply style to the same tag, and allow centralise the style in one place. Changing style becomes a much simpler process. Also it's simple enough to keep similar presentation across different part of the website for an organisation.
  • performance
    • Separating style segment directly reduce the repitition and thus the file size; thus, save some download time and speed up page rendering.

CSS is very handy for adjusting space between and within element. The box model divides the block element into 4 parts, from outside to inside are:

  1. Margin
  2. Border
  3. Padding
  4. Actual content
The below picture gives a better idea about how the box model looks like.

Margin and padding got 4 sides each (top, bottom, left, right), and can be adjusted separately.

At the right hand side is a simple demenstration about what each part represents. The number input field itself is a block element. The number inside is its child, in other word, content.

Here's a text summary about each:
  • Margin - the space outside, dictate how much distance between this element and other element.
  • Padding - the space inside, dictate how close the content is to the border.
  • Border - the outline of the block container.
